Warning Banner #1: All Items Unclassified

Source: classification_agents.py:1511-1527 — Banner injected at top of email when ALL items fail classification

CURRENT OUTPUT — Unclassified Banner (with item list)

⚠️ לא הצלחנו לסווג את הפריטים

המערכת לא הצליחה לקבוע קודי HS עבור הפריטים שלהלן. יש לספק תיאור מפורט יותר של המוצרים, כולל: חומר גלם, שימוש מיועד, הרכב.

  • Electrothermic hair-drying apparatus 1600W
  • Electric shaving machine with trimmer
  • Replacement blade cartridges (5-pack)

אנא השיבו עם מידע נוסף ונחזור עם סיווג.

Good structure. Hebrew text is correct and clear. Asks for the right information (raw material, intended use, composition). Shows the actual item descriptions so the user knows which items failed.
Minor Hebrew issue: "לא הצלחנו לסווג" — should be "לסווג" (with double vav) or preferably the more natural "לסווג" (סיווג, from root ס.ו.ג). Actually this is correct Hebrew — "לסווג" is the proper verb form. No change needed.
Note: Item list items come from invoice extraction and are typically English (invoice descriptions). This is acceptable since invoice text IS in English. Not an issue.

סווגו 3 מתוך 4 פריטים, אך רמת הוודאות הממוצעת נמוכה. מומלץ לבדוק את הסיווגים לפני שימוש. ייתכן שנדרש מידע נוסף על המוצרים.

Warning Banner #3: Clarification Needed

Source: classification_agents.py:1470-1508 — Banner when multiple HS codes are possible, user must choose

CURRENT OUTPUT — Clarification Banner (candidates table + questions)

⚠️ דרושה הבהרה לסיווג מדויק

המערכת מצאה מספר אפשרויות סיווג. נא לבחור או לספק מידע נוסף:

# פרט מכס תיאור מתעריף נימוק
א 8516.31.0000 מייבשי שיער Hair dryer — personal electro-thermic appliance
ב 8516.33.0000 מכשירי סידור שיער Hand-drying apparatus — if designed for commercial/salon use

לצורך סיווג מדויק, נא הבהירו:

  • מייבש שיער חשמלי Philips BHD300: Is this for personal/household use or professional/salon use?

נא להשיב למייל זה עם התשובות.

Good structure. Hebrew table headers are correct ("פרט מכס", "תיאור מתעריף", "נימוק"). Hebrew letter numbering (א, ב, ג) is professional. Action prompt "נא להשיב למייל זה" is clear.
ISSUE: "נימוק" column contains ENGLISH text. The reasoning comes from AI agents and is always in English ("Hair dryer — personal electro-thermic appliance"). This column appears in a Hebrew RTL table. Mixed direction is confusing.
ISSUE: Distinguishing question is in ENGLISH. "Is this for personal/household use or professional/salon use?" — the user sees Hebrew headers and labels, then suddenly an English question. Should be Hebrew: "האם המוצר לשימוש ביתי או מקצועי?"

Prioritized Recommendations (Templates Only)
1. [HIGH] Wrap error templates in RCB branding
Both _fail_html (line 2498) and _err_html (line 2554) should use a minimal version of the report template: header bar (gradient with "RCB — דו״ח סיווג"), error content in the middle, and footer with logo + disclaimer. This is a TEMPLATE change only — wrap the existing text in the same design system as the classification report.
2. [HIGH] Fix pipeline error wording
Change line 2557 from "אנחנו עובדים על כך" to honest, actionable text: "נתקלנו בשגיאה טכנית. נא לשלוח שוב את המייל והמערכת תנסה שנית." — this is a TEXT change only, no logic change.
3. [MEDIUM] Standardize error color
Change #d63384 (pink/magenta) to #991b1b (dark red) in both error templates. This matches the system design language used in warning banners, risk sections, and gap alerts. CSS-only change on lines 2500 and 2556.
4. [MEDIUM] Add diagnostic hints to extraction failure
After "לא הצלחנו לקרוא את הקבצים", add a bullet list of common causes and solutions (see proposed improvement above). This is CONTENT only — additional HTML lines in the template string, no logic change.
5. [MEDIUM] Add Hebrew reasoning to clarification banner
The reasoning field in the clarification candidates comes from AI agents and is always English. If translating the reasoning is too complex (it changes with every classification), at minimum translate the distinguishing_info question text. This would require Agent 2 to output Hebrew reasoning or a post-processing translation step — may be deferred.
